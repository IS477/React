¿Qué es React.js?
Descubre la biblioteca de JavaScript más popular del mundo para construir interfaces de usuario modernas y reactivas.

⚛️ React es una biblioteca de JavaScript de código abierto creada por Facebook (Meta) en 2013, diseñada para construir interfaces de usuario (UI) de forma declarativa, eficiente y flexible.

🧩

Basado en Componentes

Construye UI con piezas pequeñas e independientes que se combinan.

⚡

Súper Rápido

Virtual DOM actualiza solo lo que cambió, no toda la página.

🔄

Reactivo

La UI se actualiza automáticamente cuando los datos cambian.

📱

Multiplataforma

Web, móvil (React Native), desktop y más.

🌍

Gran Ecosistema

Millones de librerías y una comunidad enorme.

🛠️

Herramientas Potentes

DevTools, Next.js, Remix y más frameworks top.

🤔 ¿Por qué React y no HTML puro?

Con HTML puro, si quieres actualizar un contador en pantalla, tienes que:

  1. Buscar el elemento en el DOM con getElementById
  2. Calcular el nuevo valor manualmente
  3. Actualizar el texto del elemento
  4. Repetir esto para cada cambio

Con React, solo describes qué debe mostrar la UI y React se encarga del resto automáticamente.

❌ JavaScript Vanilla 
// Tedioso y propenso a errores
let count = 0;
const btn = document.getElementById('btn');
const display = document.getElementById('num');

btn.addEventListener('click', () => {
  count++;
  display.textContent = count;
  // Si hay más lógica, explota la complejidad
});
✅ React 
// Declarativo, simple y escalable
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicks: {count}
    </button>
  );
}

🌳 Virtual DOM — El Secreto de la Velocidad

El Virtual DOM (DOM Virtual) es una representación en memoria del DOM real del navegador. React trabaja con esta copia liviana en lugar del DOM real.

📝 JSX — JavaScript + HTML juntos

JSX (JavaScript XML) es una extensión de sintaxis que te permite escribir código similar a HTML dentro de JavaScript. No es HTML real — React lo convierte a llamadas React.createElement().

JSX (lo que escribes) 
const elemento = (
  <div className="card">
    <h1>Hola, {nombre}</h1>
    <p>Bienvenido</p>
  </div>
);
JavaScript (lo que genera React) 
const elemento = React.createElement(
  'div',
  { className: 'card' },
  React.createElement('h1', null, `Hola, ${nombre}`),
  React.createElement('p', null, 'Bienvenido')
);

💡 Reglas clave de JSX:
• Usar className en vez de class (reservada en JS)
• Todo expresión JS va entre llaves { }
• Los elementos deben tener un solo elemento raíz (o usar <>...</>)
• Las etiquetas self-closing deben cerrar: <img />, <br />

🔄 Flujo de datos — One-Way Data Flow

React usa flujo de datos unidireccional: los datos fluyen de padre a hijo mediante props. Esto hace el código predecible y fácil de depurar.

Flujo de datos con Props 
// Componente Padre (tiene los datos)
function App() {
  const usuario = { nombre: "Ana", edad: 28 };

  return <Perfil nombre={usuario.nombre} edad={usuario.edad} />;
}

// Componente Hijo (recibe los datos via props)
function Perfil({ nombre, edad }) {
  return (
    <div>
      <h2>{nombre}</h2>       // "Ana"
      <p>Edad: {edad}</p>  // "28"
    </div>
  );
}

Instalación y Configuración
Configura tu entorno de desarrollo y crea tu primer proyecto React en minutos con las herramientas modernas.

📋 Requisitos previos

  • Node.js v18 o superior — descárgalo en nodejs.org
  • npm (incluido con Node) o yarn / pnpm
  • Editor de código — VS Code (recomendado) + extensión "ES7 React/Redux Snippets"
1

Verifica tu instalación de Node.js

Abre la terminal y ejecuta estos comandos para confirmar que Node está instalado:

Terminal 
node --version    # Debe mostrar v18.x.x o superior
npm --version     # Debe mostrar 9.x.x o superior
2

Crea tu proyecto con Vite (recomendado)

Vite es el bundler más rápido y moderno. Es la forma recomendada de crear proyectos React en 2024:

Terminal 
npm create vite@latest mi-app -- --template react
# O con TypeScript (muy recomendado):
npm create vite@latest mi-app -- --template react-ts
3

Instala las dependencias

Terminal 
cd mi-app
npm install
4

Inicia el servidor de desarrollo

Terminal 
npm run dev
# Abre: http://localhost:5173

💡 ¿Por qué Vite y no Create React App (CRA)?
CRA (create-react-app) fue el estándar anterior, pero ahora está en mantenimiento mínimo. Vite es 10-100x más rápido en desarrollo, tiene mejor soporte de TypeScript y es el recomendado por el equipo de React en 2024.

Componentes en React
Los componentes son los bloques fundamentales de React. Aprende a crearlos, usarlos y combinarlos para construir interfaces complejas.

Un componente es una función de JavaScript que devuelve JSX (interfaz visual). Piensa en ellos como bloques de LEGO: pequeños, reutilizables y combinables.

🏗️ Tu primer componente

Tarjeta.jsx 
// 1. Nombre siempre con PascalCase (primera letra mayúscula)
function Tarjeta() {
  return (                        // 2. Retorna JSX
    <div className="tarjeta">
      <h2>Mi Primer Componente</h2>
      <p>¡Hola desde React! 👋</p>
    </div>
  );
}

export default Tarjeta;  // 3. Exporta el componente

// Uso en App.jsx:
import Tarjeta from './Tarjeta'

function App() {
  return <Tarjeta />  // ← Así de simple
}

📦 Props — Pasando datos al componente

Las props son como parámetros de función: permiten que un componente reciba datos desde su padre y se comporte de forma diferente según esos datos.

PerfilUsuario.jsx 
// Definición del componente con props
function PerfilUsuario({ nombre, cargo, avatar }) {
  return (
    <div className="perfil">
      <img src={avatar} alt={nombre} />
      <h3>{nombre}</h3>
      <p>{cargo}</p>
    </div>
  );
}

🔁 Renderizado de listas con .map()

Para mostrar una lista de datos, usamos .map() para transformar el array en elementos JSX. Cada elemento necesita una key única.

Lista de productos 
const productos = [
  { id: 1, nombre: "Laptop",  precio: 1200 },
  { id: 2, nombre: "Mouse",   precio: 25   },
  { id: 3, nombre: "Teclado", precio: 80   },
];

function ListaProductos() {
  return (
    <ul>
      {productos.map((producto) => (
        <li key={producto.id}>          // key es obligatorio
          {producto.nombre} — ${producto.precio}
        </li>
      ))}
    </ul>
  );
}

❓ Renderizado condicional

Condicionales en JSX 
function Saludo({ estaLogueado }) {
  // Opción 1: operador ternario
  return (
    <div>
      {estaLogueado ? (
        <h1>¡Bienvenido de vuelta! 👋</h1>
      ) : (
        <h1>Por favor, inicia sesión</h1>
      )}

      // Opción 2: operador && (cortocircuito)
      {estaLogueado && <button>Mi Perfil</button>}
    </div>
  );
}

Estado (State) con useState
El estado es el corazón de React: permite que tus componentes "recuerden" información y actualicen la UI automáticamente cuando esa información cambia.

🧠 El estado es información que el componente "recuerda" entre renders. Cuando el estado cambia, React re-renderiza el componente automáticamente mostrando los nuevos valores.

🪝 El Hook useState

Anatomía de useState 
import { useState } from 'react'

function MiComponente() {
  // [valorActual, funcionParaCambiarlo] = useState(valorInicial)
  const [count, setCount] = useState(0);
  //      ↑             ↑              ↑
  //   El dato     Función setter    Valor inicial

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Incrementar
      </button>
    </div>
  );
}

import { useState } from 'react'

function Contador() {
  const [cuenta, setCuenta] = useState(0);

  const incrementar = () => setCuenta(c => c + 1);
  const decrementar = () => setCuenta(c => c - 1);
  const reiniciar    = () => setCuenta(0);

  return (
    <div className="contador">
      <h2>{cuenta}</h2>
      <button onClick={decrementar}>−</button>
      <button onClick={reiniciar}>Reset</button>
      <button onClick={incrementar}>+</button>
    </div>
  );
}

export default Contador;

📋 Mis Tareas

    import { useState } from 'react'

function ListaTareas() {
  const [tareas, setTareas] = useState([]);
  const [texto, setTexto]  = useState('');

  const agregar = () => {
    if (!texto.trim()) return;
    setTareas([...tareas, {
      id: Date.now(),
      texto,
      hecha: false
    }]);
    setTexto('');
  };

  const completar = (id) => {
    setTareas(tareas.map(t =>
      t.id === id ? { ...t, hecha: !t.hecha } : t
    ));
  };

  const eliminar = (id) => {
    setTareas(tareas.filter(t => t.id !== id));
  };

  return (
    <div>
      <input
        value={texto}
        onChange={e => setTexto(e.target.value)}
      />
      <button onClick={agregar}>+</button>
      {tareas.map(tarea => (
        <div key={tarea.id}>
          {tarea.texto}
          <button onClick={() => eliminar(tarea.id)}>
            ✕
          </button>
        </div>
      ))}
    </div>
  );
}

    setLista([...lista, nuevo])
